# Security policy

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability. Use GitHub's
private vulnerability reporting for this repository. Include the affected
version or commit, a minimal reproduction, the expected impact, and any known
mitigations.

If private vulnerability reporting is unavailable, contact the repository
owner through the email address on the owning GitHub account and avoid sending
secrets or production data in the initial message.

## Supported versions

Until the first tagged release, security fixes are made on the `main` branch.
After releases begin, this file will list the supported release lines.

## Response

Reports will be acknowledged after they are reproduced and triaged. Fixes are
coordinated privately when disclosure before a patch would put users at risk.
